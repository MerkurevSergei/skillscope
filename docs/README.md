# SkillScope Docs

Эта папка хранит рабочий контекст проекта для разработки с Codex.

Цель: держать бизнес-решения, технические договоренности и схему данных в явном виде, чтобы новые задачи можно было делать без повторного восстановления смысла из кода.

## Документы

- [business/](business/) — бизнес-требования, use cases, user stories, роли, сценарии, ограничения и открытые вопросы.
- [development-specs.md](development-specs.md) — технические правила реализации, API-договоренности, backend/frontend conventions и Definition of Done.
- [database-schema.md](database-schema.md) — текущая схема БД, связи, индексы и правила изменения миграций.

## Как поддерживать

- Обновлять документы вместе с изменением поведения, API или схемы БД.
- Не хранить здесь временные мысли, черновики без решения и устаревшие варианты.
- Если решение еще не принято, писать его в раздел `Открытые вопросы`.
- Если код расходится с документами, считать это дефектом контекста и сначала уточнять источник истины.
